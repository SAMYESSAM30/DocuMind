# BRD Training Examples

This directory contains sample BRD documents for training and testing the analyzer.

## Files

1. **sample-brd-1.txt** - E-Commerce Platform User Authentication Module (English)
   - Comprehensive authentication system requirements
   - Multiple authentication methods
   - Security requirements
   - Frontend components

2. **sample-brd-2-ar.txt** - Order Management System (Arabic)
   - Order processing system
   - Arabic language BRD
   - Complete functional and non-functional requirements

## How to Use

1. Upload any of these files to test the analyzer
2. The system will automatically detect the language (English or Arabic)
3. All requirements will be extracted and categorized

## Adding More Training Data

To improve the analyzer, you can:
1. Add more BRD examples in this directory
2. Use consistent formatting (numbered requirements, clear sections)
3. Include both English and Arabic examples
4. Cover different domains (e-commerce, healthcare, finance, etc.)

## Format Guidelines

For best results, BRD documents should include:
- Clear section headers (numbered or titled)
- Requirement IDs (FR-001, NFR-002, etc.)
- User stories in standard format
- API endpoints documentation
- Frontend requirements with component/page references
